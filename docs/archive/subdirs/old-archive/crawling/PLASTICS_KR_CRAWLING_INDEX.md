## 💡 Key Technical Details

### HTML Structure

**Listing Page** (e.g., `?page=1`):
```html
<div class="item">
  <a href="/news/article_view.html?...">Article Title</a>
  <span class="date">2025-01-15</span>
  <span class="author">Author</span>
</div>
```

**Detail Page** (article view):
```html
<article>
  <h1 class="title">Full Title</h1>
  <div class="content">
    <p>Article paragraphs...</p>
  </div>
</article>
```

### URL Parameters

**Listing Page**:
```
/articleList.html?sc_section_code=S1N1&view_type=sm&page=1
                  └─ Category ─────────────────────  └─ Page number
```

**Article Page**:
```
/article_view.html?idx=12345&...
```

### Pagination Logic

```
Current page: 1
Next page URL: Replace page=1 with page=2
Stop when: page > max_pages OR no next link exists
```

## 🎓 Learning Value from Plasticnet Refactoring

**What Worked Well** (Reuse These):
- ✅ Pagination loop with `current_url` tracking
- ✅ URL deduplication to prevent re-crawling
- ✅ Progress JSON file for resumable crawling
- ✅ JSONL format (space-efficient, line-based)
- ✅ Batch processing with rate limiting
- ✅ Error handling with retry logic

**What We Don't Need** (Skip These):
- ❌ Legacy encoding detection (UTF-8 is standard)
- ❌ Multiple extraction strategies (one works here)
- ❌ Complex TABLE navigation (simple DIV selectors)
- ❌ HTML structure variation handling

## 📞 Questions & Answers

### Q: Why is plastics.kr simpler than plasticnet?
**A**: Modern DIV-based HTML vs legacy TABLE-based structure. UTF-8 encoding vs legacy EUC-KR. All articles follow the same format (no multiple strategies needed).

### Q: How long will the full crawl take?
**A**: ~90-110 minutes (1.5-2 hours) for 1,138 articles with 0.3s delay per article for respectful crawling.

### Q: Can I reuse code from plasticnet crawler?
**A**: Yes! Reuse: pagination loop, URL deduplication, progress tracking, JSONL format. Skip: encoding workarounds, multiple strategies, complex table parsing.

### Q: What if extraction fails on some articles?
**A**: Error handling will log failures, continue with next article, and provide summary of failed URLs in logs.

### Q: Can I resume an interrupted crawl?
**A**: Yes! `--resume` flag will skip already-crawled URLs and continue from last successful page.
