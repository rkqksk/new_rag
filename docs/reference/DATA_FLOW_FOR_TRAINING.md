## 🔄 LoRA 훈련을 위한 데이터 변환 파이프라인

```
┌────────────────────────────────────────────────┐
│  PHASE 1: 기존 RAG 데이터 기반 훈련 데이터 생성  │
│         (Colima - Local)                       │
└────────────────────────────────────────────────┘

Step 1: RAG Database에서 상담 시나리오 생성
────────────────────────────────────────
사용자 질문 (시뮬레이션)
    ↓
50미리 용기 추천해줘
제품에서 냄새가 나요
용기가 변형되었어요
...

Step 2: Teacher (7B)가 각 질문에 대해 고품질 응답 생성
────────────────────────────────────────
Question: "50미리 용기 추천해줘"
    ↓
RAG 검색: Qdrant에서 관련 문서 조회
    ↓ (검색 결과와 함께 Teacher 호출)
Teacher Response: "고품질 추천 응답..."
    ↓
RAGAS 평가: score = 0.92 ✓

Step 3: 필터링 (score >= 0.80)
────────────────────────────────────────
High Quality (>0.80)  → 훈련 데이터 사용
Low Quality  (<0.80)  → 폐기 또는 재처리

Step 4: 훈련 데이터 포맷 변환
────────────────────────────────────────
{
  "input": "50미리 용기 추천해줘\n[검색 결과]\n문서1: 50ml PET...\n문서2: 50ml HDPE...",
  "output": "Teacher가 생성한 고품질 응답...",
  "metadata": {
    "ragas_score": 0.92,
    "sources": ["sample_products.txt"],
    "original_query": "50미리 용기 추천해줘",
    "embedding_model": "sentence-transformers/all-MiniLM-L6-v2",
    "consultation_type": "product_recommendation"
  }
}

Step 5: 최종 훈련 데이터셋 생성
────────────────────────────────────────
training_data.json (리스트 형식)
[
  {훈련 샘플 1},
  {훈련 샘플 2},
  ...
  {훈련 샘플 N}
]

총 샘플: 예상 200-500개 (RAG DB 크기에 따라)
```

## 🔑 핵심 포인트: RAG 데이터 활용

### RAG Database의 역할

```
RAG Database (Qdrant)
│
├─ 1️⃣ 검색 데이터 제공
│  "50미리 용기 추천" → Qdrant 검색 → Top-5 문서 반환
│
├─ 2️⃣ 훈련 데이터 생성의 기초
│  Teacher 응답 생성 시 RAG 검색 결과를 프롬프트에 포함
│  → 훈련 데이터도 같은 검색 결과 포함
│
└─ 3️⃣ 일관성 보장
   Student 모델도 동일한 RAG 검색 결과를 기반으로 학습
   → 프로덕션에서도 같은 방식으로 작동
```

### 중요: 컨텍스트 포함

```
❌ 잘못된 방식 (컨텍스트 없음)
─────────────────────────────
Input: "50미리 용기 추천해줘"
Output: "50ml 용기를 추천합니다"
→ Student가 RAG 없이 학습 (배포 시 불일치)

✅ 올바른 방식 (RAG 검색 결과 포함)
─────────────────────────────
Input: """
50미리 용기 추천해줘

[검색 결과]
- 50미리 용기 (Standard): PET 투명, 1,000원/개
- 50미리 용기 (Opaque): HDPE 불투명, 1,200원/개
"""
Output: "PET 투명 용기를 추천합니다. 가성비가 좋고..."
→ Student가 RAG 컨텍스트와 함께 학습 (배포 시 일치)
```

## 💡 핵심 이해: RAG 기반 LoRA Training

### 왜 RAG 데이터를 훈련에 포함하는가?

```
배포 환경:
┌─────────────────────────────┐
│ User Query                  │
│   ↓                         │
│ RAG 검색 (Qdrant)          │
│   ↓                         │
│ 검색 결과 + Student 응답     │
└─────────────────────────────┘

따라서 Student 모델은:
✅ 훈련: RAG 검색 결과와 함께 학습
✅ 배포: RAG 검색 결과와 함께 사용
→ 학습 방식 = 배포 방식 (일관성)
```

### LoRA 훈련 설정 (Linux)

```python
# training_config.json
{
  "model_name": "qwen2.5:3b",
  "training_data": "./data/training_data.json",
  "lora_config": {
    "r": 16,
    "lora_alpha": 32,
    "target_modules": ["q_proj", "v_proj"],
    "lora_dropout": 0.05
  },
  "training_args": {
    "num_epochs": 3,
    "per_device_train_batch_size": 8,
    "learning_rate": 1e-4,
    "warmup_steps": 100,
    "max_steps": 2000
  },
  "evaluation": {
    "compare_with_teacher": true,
    "metrics": ["RAGAS", "accuracy", "latency"]
  }
}
```

```
┌─ Qdrant에 있는 데이터

│  Q1: "50미리 용기 추천해줘"

│  Q3: "용기가 변형되었어요"
│
├─ Step 2: Teacher 응답 (각 질문마다)
│  Q1 + RAG검색 → Teacher → "PET 투명 용기를 추천합니다..."
│  Q2 + RAG검색 → Teacher → "포장재 냄새입니다. 환기하세요..."
│  Q3 + RAG검색 → Teacher → "배송 파손이 의심됩니다..."
│
├─ Step 3: RAGAS 평가
│  응답1: 0.92 ✓ (포함)
│  응답2: 0.78 ✗ (제외)
│  응답3: 0.85 ✓ (포함)
│
├─ Step 4: 훈련 데이터 2개 생성
│  {
│    "input": "50미리 용기 추천해줘\n[검색결과]...",
│    "output": "PET 투명 용기를..."
│  },
│  {
│    "input": "용기가 변형되었어요\n[검색결과]...",
│    "output": "배송 파손이..."
│  }
│
└─ Step 5: training_data.json으로 LoRA 훈련
   → Student 모델이 2개 샘플로 학습
   → 제조업 상담에 특화된 모델 생성
```

## ⚠️ 주의사항

### 1. 데이터 크기
```
현재: 1개 문서 → ~3개 훈련 샘플 (너무 작음)
목표: 50-100개 문서 → 200-500개 훈련 샘플 (적절)

따라서:
- 제조업 문서를 더 추가해야 함
- 또는 시뮬레이션으로 다양한 질문 생성
```

### 2. 품질 필터링
```
RAGAS score >= 0.80 필수
- 0.80 이상: 훈련에 사용
- 0.80 미만: 버림 또는 Teacher 재검토

이렇게 해야 Student가 고품질 데이터만 학습
```

### 3. 일관성 유지
```
훈련 데이터에 포함된 검색 결과
= 배포 환경의 검색 결과
→ 모델이 배포 환경에 최적화됨
```
