### BUG-001: Docker 컨테이너 간 네트워크 연결 불안정

**날짜**: 2024-12-01  
**우선순위**: 높음  
**상태**: ✅ 해결됨  
**담당자**: DevOps Team  
**해결일**: 2024-12-03

#### 증상
Qdrant 컨테이너가 간헐적으로 FastAPI에서 접근 불가

#### 재현 단계
1. `docker-compose up -d` 실행
2. FastAPI에서 `/search` 엔드포인트 호출
3. 10% 확률로 연결 타임아웃

#### 예상 동작
항상 Qdrant에 연결 가능해야 함

#### 실제 동작
간헐적으로 `ConnectionError: Failed to connect to Qdrant`

#### 환경
- OS: macOS 14.0
- Docker: 24.0.6
- Docker Compose: 2.23.0

#### 해결 방법
1. `docker-compose.yml`에 `depends_on` 조건 추가
2. Health check 구현
3. Retry 로직 추가 (최대 3회, 지수 백오프)

```yaml
services:
  fastapi:
    depends_on:
      qdrant:
        condition: service_healthy
```

#### 테스트
- [x] 100회 연속 호출 테스트 통과
- [x] 재시작 후에도 정상 동작 확인

### BUG-002: 임베딩 생성 시 메모리 누수

**날짜**: 2024-12-10  
**우선순위**: 중간  
**상태**: ✅ 해결됨  
**담당자**: ML Team  
**해결일**: 2024-12-12

#### 증상
대량 문서 임베딩 시 메모리 사용량이 계속 증가하여 OOM 발생

#### 재현 단계
1. 1000개 문서 배치 임베딩 시작
2. 메모리 사용량 모니터링
3. 500개 처리 후 메모리 8GB → 16GB 증가

#### 예상 동작
배치 처리 후 메모리 해제

#### 실제 동작
메모리가 계속 누적되어 OOM 발생

#### 환경
- OS: macOS 14.0
- Python: 3.11.6
- Transformers: 4.36.0

#### 해결 방법
1. 배치 처리 후 명시적 `del` 및 `gc.collect()`
2. 토치 캐시 클리어 `torch.cuda.empty_cache()`
3. 배치 크기 조정 (32 → 16)

```python
for batch in batches:
    embeddings = model.encode(batch)
    store_embeddings(embeddings)
    del embeddings
    gc.collect()
    torch.cuda.empty_cache()
```

#### 테스트
- [x] 1000개 문서 처리 시 메모리 안정적 유지
- [x] 메모리 프로파일링으로 누수 없음 확인

### BUG-003: 한글 검색 시 BM25 정확도 저하

**날짜**: 2024-12-18  
**우선순위**: 높음  
**상태**: ✅ 해결됨  
**담당자**: ML Team  
**해결일**: 2024-12-20

#### 증상
한글 질의 시 BM25 검색 결과가 부정확

#### 재현 단계
1. "PE 재질 50ml 용기" 검색
2. BM25 결과 확인
3. 관련 없는 문서가 상위 랭킹

#### 예상 동작
"PE", "재질", "50ml", "용기" 키워드가 모두 포함된 문서 우선

#### 실제 동작
형태소 분리 없이 공백으로만 토큰화되어 매칭 실패

#### 환경
- Python: 3.11.6
- BM25: rank-bm25==0.2.2

#### 해결 방법
1. Mecab 형태소 분석기 통합
2. 불용어 사전 추가
3. 복합어 처리 로직 구현

```python
from konlpy.tag import Mecab

mecab = Mecab()

def tokenize_korean(text):
    tokens = mecab.morphs(text)
    return [t for t in tokens if t not in stopwords]
```

#### 테스트
- [x] 한글 질의 정확도 85% → 92% 향상
- [x] 100개 테스트 케이스 통과

## 📝 버그 리포팅 가이드

### 좋은 버그 리포트의 요소
1. **명확한 제목**: 문제를 한 줄로 요약
2. **재현 단계**: 누구나 재현 가능하도록 상세히 기술
3. **환경 정보**: OS, 브라우저, 버전 등
4. **스크린샷/로그**: 가능한 경우 첨부
5. **예상/실제 동작**: 무엇이 잘못되었는지 명확히

### 버그 우선순위 기준

**긴급 (Critical)**
- 시스템 다운
- 데이터 손실
- 보안 취약점

**높음 (High)**
- 주요 기능 동작 불가
- 성능 심각한 저하
- 다수 사용자 영향

**중간 (Medium)**
- 부분 기능 동작 불가
- 회피 방법 존재
- 일부 사용자 영향

**낮음 (Low)**
- UI/UX 개선
- 문서 오류
- 최소 영향
